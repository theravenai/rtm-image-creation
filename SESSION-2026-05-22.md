# Image Studio — Session Summary
**Date:** May 22–23, 2026

---

## Bugs Fixed

### 1. Pool Image Selection (Critical)
**Problem:** Clicking any pool image in the Asset Library failed silently — `InternalServerError` every time.

**Root cause:** `get_asset` was called in the pool regen code path inside `sessions.py` but was never imported from `..storage`. This caused a `NameError` at runtime.

**Fix:** Added `get_asset` to the import block in `api/routers/sessions.py`.

---

### 2. Ghost Processes Blocking Port 8000
**Problem:** Two unkillable zombie processes (PIDs 595724 and 509524) were holding port 8000 with old pre-fix code. Even after patching the import, ~50% of requests hit the ghost processes and returned `[WinError 233] No process is on the other end of the pipe`.

**Fix:** Started a clean server on port 8001, added `NEXT_PUBLIC_IMAGE_STUDIO_API_URL=http://localhost:8001` to `.env.local`, restarted Next.js dev server (which auto-picked port 3001 since 3000 was taken by another app).

---

### 3. Gemini Regen Broken When Section Source is "Pexels"
**Problem:** Clicking "Rerun" or any reprompt chip on a section that had previously used Pexels would re-fetch a Pexels image instead of generating a Gemini AI image. The backend defaulted to `sec.get("source", "gemini")` — so if `source="pexels"` was stored, Pexels was used.

**Fix:** `handleRegenerate` in `page.tsx` now always sends `source: "gemini"` explicitly in the request body. Rerun and chips always produce AI images regardless of what source was previously stored.

```typescript
// Before
const body: Record<string, unknown> = {};

// After
const body: Record<string, unknown> = { source: "gemini" };
```

## Features Built

### 4. Pool Regen — Apply Pool Image to Any Section
**What:** Clicking a pool image in the Asset Library now correctly applies it to the target section — copies the asset file as the background, composites text overlay, and returns `composited_url` immediately.

**Backend (`sessions.py`):**
- Full `pool_image_id` code path in `regen_section`: looks up asset by ID, copies file to session directory, calls `composite_section`, returns `{ section_number, background_url, composited_url, source: "pool", status: "done" }`

**Frontend (`assets-library.tsx`):**
- New `onImageApplied(sectionNumber, data)` callback replaces old `onImageSelected`
- Parses full regen response and passes to `handleSectionUpdate` so the section card updates immediately with the new image — no stale state, no stuck "generating" spinner

---

### 5. Save to Pool — Background Only (No Text Overlay)
**What:** Pool and Save BG buttons now save the raw background image without any composited text, so pool images remain reusable across different sections with different text.

**Logic:**
- `image_type: "background"` → saves `background_path` (AI-generated raw image)
- `image_type: "pexels_bg"` → saves `pexels_background_path` (raw Pexels photo)
- Never saves the composited version with text baked in

---

### 6. Pexels Cache-Busting
**What:** Pexels alternative images were serving stale cached versions when searching again (same filename `pexels_composited.png`).

**Fix:** Appended `?cb=${Date.now()}` to the Pexels URL on every new fetch result.

---

### 7. Reprompt Chips — Full Prompt Template
**What:** Chips were sending just the short description ("Piggy bank with Canadian currency") directly to Gemini, producing generic results.

**Fix:** All chip clicks now wrap the description in a full photorealistic prompt template:

```
Photorealistic, professional editorial photography for a Canadian mortgage
and real estate article. [chip description]. Wide 16:9 aspect ratio, natural
lighting, rule of thirds composition, no text overlays, no watermarks, no logos.
Clean, trust-inspiring, suburban or urban Toronto/Ottawa setting.
```

---

### 8. Background Download Button
**What:** Added a "BG" download link on each section card that downloads the raw background image (no text overlay) directly from the browser.

---

### 9. Section Picker in Asset Library
**What:** When the Asset Library is opened from the sidebar (no pre-selected section), users can now pick which section to apply the pool image to — inline dropdown in the library header showing all section numbers and H2 titles.

---

### 10. Asset Pool Categories / Folders
**What:** Pool images can now be organized into named folders (e.g. "Mortgage Approval", "Home Buying", "Refinancing").

**Backend changes:**
- `category` field added to all pool asset records in `pool.json`
- `GET /assets/categories` — returns sorted list of unique category names
- `POST /sessions/{id}/sections/{n}/save-to-pool` — accepts `category` in body
- `PUT /assets/pool/{id}` — accepts `category` as query param for updates

**Frontend — section-card.tsx:**
- Clicking "Pool" or "Save BG" opens an inline category picker strip (yellow panel)
- Autocomplete from existing categories via `<datalist>`
- Existing categories shown as clickable chips for quick selection
- Category name included in success toast: _"Saved to pool → Mortgage Approval"_

**Frontend — assets-library.tsx:**
- Folder filter row appears above tag pills when categories exist
- Clicking a folder filters images to that category only
- Tag pills scope to images within the selected folder
- Category resets to "All" each time the library opens
